mws

GitHub Releases Issues

Multi-task Workspace Manager — like git worktree, but for your entire multi-repo project.

Synopsis

$ curl -fsSL https://raw.githubusercontent.com/paulo20223/mws/main/install.sh | bash

Description

mws creates fully isolated copies of your multi-repo workspace so you can work on multiple tasks in parallel without conflicts. Each task gets its own copy of every repo, config file, and docker-compose — with automatic branch switching across all git repositories.

When your project spans multiple repos, shared configs, and compose files, switching tasks means stashing changes across repos, hoping configs don't conflict, and losing your flow. git worktree only handles a single repo. mws handles the whole workspace.

Quick Start

$ cd /path/to/your/workspace
$ mws init
$ mws create feature-x -b feature/x -r "backend*,frontend"
$ mws create bugfix-y -b fix/y
$ mws list
$ mws status feature-x
$ cd $(mws cd feature-x)
$ mws sync feature-x
$ mws delete bugfix-y --force

Commands

Command Description
mws initInitialize workspace, discover all git repos
mws create <name> [-b branch] [-r globs]Full workspace copy, switch branches
mws listList all tasks with sizes and branches
mws status [name]Show repo branches, modified/staged/untracked
mws delete <name> [--force]Remove task copy and metadata
mws cd <name>Print task path (use with cd $(...))
mws open <name>Open task workspace in editor
mws basePrint base workspace path
mws sync <name> [--dry-run]Sync non-repo files from base
mws shell-initOutput shell integration code

How It Works

mws create copies the entire workspace directory alongside the original, switching each git repo to the specified branch. Non-repo files (docker-compose, configs, scripts) get their own independent copies. 

# base (original)
/path/to/workspace/
├── repo-a/
├── repo-b/
├── docker-compose.yml
├── shared-config/
└── .mws/
    ├── config.toml
    └── tasks/
        └── feature-x/
            └── manifest.toml

# full isolated copy
/path/to/workspace--feature-x/
├── repo-a/                           # on branch feature/x
├── repo-b/                           # on branch feature/x
├── docker-compose.yml                # own copy, edit freely
├── shared-config/                    # own copy
└── .mws-task                         # marker file

Configuration

Config lives at .mws/config.toml in the base workspace:

[workspace]
base = "/path/to/workspace"
editor = "code"

[copy]
exclude = [
    "node_modules/",
    ".venv/",
    "__pycache__/",
    ".next/",
    "dist/",
    "build/",
    ".mws/",
    "*.pyc",
    ".DS_Store",
    "plans/",
    ".git/lfs/",
    "vendor/",
    "target/",
]

Shell Integration

Add to your ~/.bashrc or ~/.zshrc:

$ eval "$(mws shell-init)"

This gives you:

  • mcd <task> — navigate directly into a task workspace
  • mbase — navigate back to the base workspace
  • [mws:task-name] in your shell prompt when inside a task

Install

curl (recommended)

$ curl -fsSL https://raw.githubusercontent.com/paulo20223/mws/main/install.sh | bash

Homebrew

$ brew install paulo20223/tap/mws

Debian / Ubuntu

$ curl -fsSLO https://github.com/paulo20223/mws/releases/latest/download/mws_amd64.deb
$ sudo dpkg -i mws_amd64.deb

From source

Requires Go 1.22+.

$ go install github.com/paulo20223/mws/cmd/mws@latest

Manual

Download the binary for your platform from GitHub Releases and place it in your $PATH.
